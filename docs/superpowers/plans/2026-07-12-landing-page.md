# ccstatus Landing Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single-page, static, dark terminal-styled marketing site for `ccstatus` in `site/`, with an interactive braille-accurate terminal demo and full SEO/GEO/AEO.

**Architecture:** Next.js 16 App Router with `output: 'export'` (pure static). One client component drives interactivity (preset recolor, braille/block toggle, install tabs, copy); everything else is server-rendered static. The braille bar geometry is generated by an algorithm ported verbatim from `src/render.rs` (ADR-0001) so the demo matches the binary glyph-for-glyph.

**Tech Stack:** Next.js 16, React 19, Tailwind CSS v4, TypeScript, JetBrains Mono (self-hosted via `next/font/local`), deployed on Vercel.

**Spec:** `docs/superpowers/specs/2026-07-12-landing-page-design.md`
**Glossary:** `CONTEXT.md` · **ADR:** `docs/adr/0001-port-braille-algorithm-to-web.md`
**Mockup reference:** https://claude.ai/code/artifact/75228b35-d855-4d5e-9fc7-fcda7c3c097f

## Global Constraints

- Package manager: **pnpm** (10.x). All commands run from `site/`.
- Static export only: `output: 'export'` — no server functions, no runtime, no `middleware`.
- **Zero external network requests** at runtime — self-host fonts, inline/local assets only. No third-party scripts, no analytics in v1.
- Dark theme only (deliberate terminal commit). No light theme.
- Canonical URL from `NEXT_PUBLIC_SITE_URL`, default `https://ccstatus.vercel.app`.
- Accent = Claude orange `#d7875f`. Neutrals carry a warm bias (values in Task 3).
- Copy must use canonical terms from `CONTEXT.md` ("status line" two words, "braille bar", "rate-limit window", "preset", "plan label", "cold-start cache").
- All product facts must match the CLI. Install commands: `brew install morsechimwai/tap/ccstatus`, `npm install -g ccstatus-cli`, `cargo install ccstatus`.
- Motion 150–300ms, `transform`/`opacity` only, gated behind `prefers-reduced-motion`.
- Accessibility: keyboard-operable interactives, visible focus rings, contrast ≥ 4.5:1, semantic headings h1→h2→h3.

```ts
// Braille bar geometry, ported verbatim from src/render.rs `braille_bar`.
// KEEP IN SYNC with src/render.rs — see docs/adr/0001-port-braille-algorithm-to-web.md.
// filled = (pct.min(100) * 2*width + 50) / 100  (integer division)
// Each cell: baseline dots 7,8 (0xC0); left sub-col adds 0x07, right adds 0x38.

export type Cell = { glyph: string; filled: boolean };

const clampPct = (pct: number) => Math.max(0, Math.min(100, Math.trunc(pct)));

export function filledSubcols(pct: number, width: number): number {
  const subcols = 2 * width;
  return Math.floor((clampPct(pct) * subcols + 50) / 100);
}

export function brailleCells(pct: number, width = 12): Cell[] {
  const filled = filledSubcols(pct, width);
  const cells: Cell[] = [];
  for (let cell = 0; cell < width; cell++) {
    const left = 2 * cell < filled;
    const right = 2 * cell + 1 < filled;
    let bits = 0xc0;
    if (left) bits |= 0x07;
    if (right) bits |= 0x38;
    cells.push({ glyph: String.fromCodePoint(0x2800 + bits), filled: left || right });
  }
  return cells;
}

// Block-bar mode: filled = (pct * width + 50) / 100  (matches render.rs filled_cells).
export function blockCells(pct: number, width = 12): Cell[] {
  const filled = Math.floor((clampPct(pct) * width + 50) / 100);
  return Array.from({ length: width }, (_, i) => ({
    glyph: i < filled ? "█" : "░",
    filled: i < filled,
  }));
}
```
